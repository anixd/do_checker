# Справка по DO Checker

Добро пожаловать в DO Checker. Это приложение предназначено для проверки доступности сайтов через прокси-сервис SOAX.

## Основные разделы

### 1. Checker

Основная страница для запуска проверок.

* **URL(s)**: Введите один или несколько сайтов (каждый с новой строки).
* **Geo Parameters**: Выберите страну (опционально -- регион, город, ISP) для проверки. Страна обязательна.
* **Proxy Settings**:
    * `Protocol`: HTTP или SOCKS5 (для SOAX Port-режима обычно используется `http`).
    * `DNS`: Рекомендуется `via proxy`, чтобы DNS-запрос также шел через прокси.
* **Advanced / Overrides**: Позволяет вручную задать хост и порт прокси, а также включить скриншоты.

### 2. DNS Tools

Инструмент для проверки DNS-записей и определения хостинг-провайдера домена.

* **Domain(s)**: Введите домены для проверки.
* **Результат**: Показывает IP-адреса, Geo (по первому IP) и владельца (определяется по `provider_keywords` в `app.yaml`).
* **Details**: Ссылка на сырой Whois/RDAP ответ.

### 3. Geo Catalog

Управление локальным кэшем Geo-данных SOAX.

**Важно**: При первом запуске приложения этот каталог, скорее всего, не содержит данных (регионов, городов).

1.  **Manage Country List**: Убедитесь, что в списке "Keep/Remove" есть нужные вам страны. Вы можете добавить новые (ISO-2 коды) в поле "Add new countries". Нажмите **"Save List Changes"**.
2.  **Refresh Data**: Нажмите **"Refresh from SOAX (in background)"**. Это запустит фоновый процесс загрузки актуальных регионов, городов и ISP для стран из вашего списка. Процесс занимает 1-2 минуты.

### 4. Settings

Прямое редактирование файла конфигурации `data/config/app.yaml`.

**Внимание!** Изменяйте эти настройки, только если понимаете, что делаете. Неправильные/некорректные значения могут сломать приложение.

* `default_theme`: Тема по умолчанию (`auto`, `light`, `dark`).
* `http_client:custom_headers`: Добавление кастомных HTTP-заголовков ко всем запросам.
* `dns_checker:provider_keywords`: Настройка ключевых слов для определения хостера на странице **DNS Tools**.

**Приоритет настроек**: Значения из `.env` (например, `APP_PORT`) всегда имеют приоритет над значениями в `app.yaml`.
